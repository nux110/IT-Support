## Project Structure

```
├── helpdesk.html          # Main HTML file
├── admin.html             # Admin dashboard HTML
```

## Quick Start

### Prerequisites

- Node.js (v14 or higher)
- npm or yarn
- Supabase account (free tier available)

### Installation

1. **Clone or navigate to the project directory**
   ```bash
   cd "c:\Users\Miguel\Downloads\it support"
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up Supabase Database**
   - Go to [supabase.com](https://supabase.com) and create a free account
   - Create a new project
   - Go to Settings → API to get your project URL and anon key
   - Copy `supabase-setup.sql` content and run it in your Supabase SQL Editor

4. **Configure Environment Variables**
   - Copy `.env.example` to `.env` (if it exists) or create `.env`
   - Add your Supabase credentials:
   ```
   SUPABASE_URL=your_supabase_project_url
   SUPABASE_ANON_KEY=your_supabase_anon_key
   ```

5. **Start the server**
   ```bash
   npm start
   ```

6. **Open your browser**
   ```
   http://localhost:3000
   ```

### Development Mode

For development with auto-restart:
```bash
npm run dev
```

## Supabase Setup Tutorial

This tutorial will guide you through setting up Supabase for your IT support system. Supabase provides a PostgreSQL database with real-time capabilities and an easy-to-use API.

### Step 1: Create a Supabase Account

1. **Visit Supabase**
   - Go to [supabase.com](https://supabase.com)
   - Click "Start your project" or "Sign up"

2. **Sign Up Options**
   - **GitHub**: Recommended - fastest signup
   - **Email**: Traditional email/password
   - **SSO**: For enterprise users

3. **Verify Your Account**
   - Check your email for verification link
   - Complete any additional verification steps

### Step 2: Create Your First Project

1. **Access Dashboard**
   - After signup, you'll be taken to your dashboard
   - Click "New project" button

2. **Project Details**
   - **Name**: `techdesk-support` (or your preferred name)
   - **Database Password**: Choose a strong password (save this!)
   - **Region**: Select the region closest to your users
     - For US: `West US (Oregon)` or `East US (Ohio)`
     - For Europe: `West EU (Ireland)`
     - For Asia: `Southeast Asia (Singapore)`

3. **Wait for Setup**
   - Project creation takes 2-3 minutes
   - You'll see progress indicators for database setup

### Step 3: Set Up Database Schema

1. **Access SQL Editor**
   - In your project dashboard, click "SQL Editor" in the left sidebar
   - Click "New query" button

2. **Run Setup Script**
   - Copy the entire contents of `supabase-setup.sql` from your project
   - Paste it into the SQL Editor
   - Click "Run" button

3. **Verify Table Creation**
   - You should see "Success" message
   - Check "Table Editor" to see the `tickets` table
   - Verify columns: id, name, email, department, category, priority, subject, description, status, created_at, updated_at

### Step 4: Get API Credentials

1. **Access API Settings**
   - In your project dashboard, click "Settings" in the left sidebar
   - Click "API" in the settings menu

2. **Copy Required Values**
   - **Project URL**: Copy the full URL (e.g., `https://abcdefghijklmnop.supabase.co`)
   - **anon/public key**: Copy the `anon public` key (starts with `eyJ...`)

3. **Configure Environment Variables**
   - Open your `.env` file in the project root
   - Replace the placeholder values:
   ```env
   SUPABASE_URL=https://your-project-id.supabase.co
   SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```

### Step 5: Test Database Connection

1. **Start Your Server**
   ```bash
   npm start
   ```

2. **Check Console Output**
   - Look for: "🗄️ Database: Supabase PostgreSQL"
   - If you see connection errors, double-check your `.env` values

3. **Test Health Endpoint**
   - Visit: `http://localhost:3000/api/health`
   - Should return: `{"status":"healthy","database":"connected"}`

### Step 6: Test Full Application

1. **Open Application**
   - Visit: `http://localhost:3000`
   - Fill out and submit a test ticket

2. **Verify Data Storage**
   - Check Supabase dashboard → Table Editor → tickets table
   - Your submitted ticket should appear in the database

3. **Test Admin Features**
   - Visit: `http://localhost:3000/api/tickets`
   - Should return JSON with your ticket data

### Troubleshooting

#### "Invalid supabaseUrl" Error
- Check that your `SUPABASE_URL` starts with `https://`
- Ensure there are no extra spaces or characters

#### "Database connection failed" Error
- Verify your `SUPABASE_ANON_KEY` is correct
- Check that your project isn't paused (Supabase pauses inactive projects)
- Ensure Row Level Security policies are set up (they are in our SQL script)

#### Table Doesn't Exist Error
- Make sure you ran the `supabase-setup.sql` script in SQL Editor
- Check for any SQL errors when running the setup script

#### CORS Errors
- Supabase handles CORS automatically for web applications
- If you see CORS errors, check your Supabase project settings

### Security Notes

- **Never commit `.env` file** to version control
- **Use environment variables** for all sensitive data
- **Row Level Security** is enabled on the tickets table
- **anon key is safe** for client-side use (it's public)

## Netlify Deployment

Deploy your TechDesk application to Netlify for free hosting with automatic deployments.

### Prerequisites

- Supabase database set up (see tutorial above)
- GitHub account (for automatic deployments)
- Netlify account (free tier available)

### Step 1: Prepare for Deployment

1. **Build the application**
   ```bash
   npm run build
   ```

2. **Verify build output**
   - Check that `dist/` directory contains your frontend files
   - Verify `netlify/functions/` contains the API function

### Step 2: Push to GitHub

1. **Create a GitHub repository**
   ```bash
   git init
   git add .
   git commit -m "Initial commit: TechDesk IT Support Portal"
   git branch -M main
   git remote add origin https://github.com/yourusername/your-repo-name.git
   git push -u origin main
   ```

### Step 3: Deploy to Netlify

1. **Connect to Netlify**
   - Go to [netlify.com](https://netlify.com)
   - Sign up/login with GitHub
   - Click "New site from Git"

2. **Import from GitHub**
   - Choose your repository
   - Branch: `main`
   - Build command: `npm run build`
   - Publish directory: `dist`

3. **Set Environment Variables**
   - In Netlify dashboard, go to Site settings → Environment variables
   - Add your Supabase credentials:
     ```
     SUPABASE_URL=https://your-project-id.supabase.co
     SUPABASE_ANON_KEY=your-anon-key
     ```

4. **Deploy**
   - Click "Deploy site"
   - Wait for deployment to complete (2-3 minutes)

### Step 4: Configure Custom Domain (Optional)

1. **Add custom domain**
   - In Netlify dashboard: Site settings → Domain management
   - Add your custom domain or use the free `.netlify.app` subdomain

2. **Update DNS** (if using custom domain)
   - Point your domain's DNS to Netlify's nameservers
   - Netlify provides specific instructions for your domain registrar

### Step 5: Test Your Deployment

1. **Visit your site**
   - Use the URL provided by Netlify (e.g., `https://amazing-site.netlify.app`)

2. **Test functionality**
   - Submit a test ticket
   - Check that data saves to Supabase
   - Verify API endpoints work: `https://yoursite.netlify.app/api/health`

### Netlify Features Used

- **Static Site Hosting**: Frontend files served globally
- **Serverless Functions**: API routes handled by Netlify Functions
- **Automatic Deployments**: Deploy on every git push
- **Environment Variables**: Secure configuration management

## API Documentation

### Submit a Ticket
```http
POST /api/tickets
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john.doe@company.com",
  "department": "IT",
  "category": "Hardware Problem",
  "priority": "high",
  "subject": "Laptop won't start",
  "description": "My laptop stopped working after the last update..."
}
```

**Response:**
```json
{
  "ticketId": "TICK-1643123456789-A1B2C",
  "message": "Ticket submitted successfully",
  "status": "open"
}
```

### Get All Tickets
```http
GET /api/tickets
```

**Response:**
```json
{
  "tickets": [...],
  "total": 25,
  "open": 12
}
```

### Get Specific Ticket
```http
GET /api/tickets/TICK-1643123456789-A1B2C
```

### Update Ticket Status
```http
PATCH /api/tickets/TICK-1643123456789-A1B2C
Content-Type: application/json

{
  "status": "in-progress"
}
```

**Valid statuses:** `open`, `in-progress`, `resolved`, `closed`

### Health Check
```http
GET /api/health
```

## Data Storage

Uses **Supabase PostgreSQL** database with real-time capabilities:

- **Database**: PostgreSQL via Supabase
- **Real-time**: Live updates for ticket status changes
- **Security**: Row Level Security (RLS) enabled

### Database Schema

The `tickets` table includes:
- `id`: Unique ticket identifier
- `name`, `email`: User information
- `department`, `category`: Organization details
- `priority`: low, medium, high, critical
- `subject`, `description`: Ticket content
- `status`: open, in-progress, resolved, closed
- `created_at`, `updated_at`: Timestamps
